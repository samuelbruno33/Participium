# Task 3 - Use Cases

## Deliverable to produce
Complete the template file located at:

**`../doc/deliverable/03_UseCases.md`**

You must:
- link the use case diagram image,
- include the exported JSON source of the diagram under `../data/`,
- complete the use case narratives,
- complete the Traceability Table mapping use case IDs to requirement IDs.

You may add short notes, but **do not remove or rename** the existing section headings.

## General guidance

### Keep it coherent
The diagram and the narratives must describe the same system behaviour and use consistent actors, names, and terminology.

Every use case shown in the diagram must have a corresponding narrative.

### Focus on user goals
Model meaningful actor goals, not interface clicks, pages, or internal technical functions.

### Keep it clear
Prefer a compact and readable diagram over an overloaded one. Use `include`, `extend`, and generalization only when they really improve clarity.

---

## 1) Use Case Diagram

### What you must provide
Create a use case diagram and attach it as an image under `../data/img/`, then link it in the deliverable file.

Also include the editable JSON source file from UML Modeler under `../data/`.

### Expectations
The diagram must:
- show the system boundary,
- identify the relevant actors,
- represent the main use cases in scope,
- use clear verb-phrase names,
- remain readable as a single coherent model.

---

## 2) Use Case Narratives

### What you must provide
Write detailed use case narratives using the structure already present in the template.

Provide one narrative for each use case shown in the diagram.


### Expectations
Narratives should:
- be written at user-goal level,
- describe the normal success flow as numbered steps,
- include relevant extensions or alternative flows,
- define meaningful preconditions, minimum guarantees, and success guarantees.

Keep the narratives concise but complete.

---

## 3) Traceability Table

### What you must provide
Complete the Traceability Table by mapping use case IDs to the related requirement IDs.

Use one row per mapping.
