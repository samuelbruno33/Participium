## General guidance

### Keep it coherent
The diagrams must use the same terminology, actors, system elements, and assumptions already introduced in the previous deliverables.

### Keep it readable
Prefer compact and understandable diagrams over overloaded ones. Include the elements that matter for communication and traceability, and avoid unnecessary low-level detail.

### Use the diagrams as design artifacts
The class diagram should clarify the main concepts of the system and their relationships. The deployment diagram should clarify how the system is physically/logically deployed and which external services it depends on.
