# Participium Frontend

React/Vite frontend for Participium. The frontend is separated from the Flask backend, does not access the database directly, and communicates with the backend only through the REST API.

## Stack

- React 19
- TypeScript
- Vite
- React Router
- Leaflet for OpenStreetMap views

## Prerequisites

Install Node.js LTS. Node.js includes `npm`.

### Windows

Using Windows Package Manager:

```powershell
winget install OpenJS.NodeJS.LTS
```

Verify the installation:

```powershell
node --version
npm --version
```

### Unix / macOS

Use the package manager available on the machine.

Example for Debian/Ubuntu:

```bash
sudo apt update
sudo apt install -y nodejs npm
```

Verify the installation:

```bash
node --version
npm --version
```

## Configuration

The frontend reads its runtime configuration from Vite environment variables.

Default values are provided in `.env.example`:

```text
VITE_API_BASE_URL=http://localhost:5050/api/v1
VITE_BACKEND_BASE_URL=http://localhost:5050
```

The local backend documented in `src/backend/README.md` runs on:

```text
http://localhost:5050
```

Create a local `.env` file only when custom values are needed.

### Windows

```powershell
Copy-Item .env.example .env
```

### Unix / macOS

```bash
cp .env.example .env
```

## Local Execution

Open a terminal in `src/frontend`.

### Windows

```powershell
npm install
npm run dev
```

### Unix / macOS

```bash
npm install
npm run dev
```

The application is available at:

```text
http://localhost:5173
```

## HTML Identifier Contract

The frontend exposes stable HTML `id` attributes on rendered application elements.

Conventions:

- static elements use descriptive IDs, for example `login-submit`, `public-filter-form`, and `admin-users-table`;
- data-generated elements use constant prefixes defined in `src/domIds.ts`, for example `public-report-row-<id>`, `assigned-report-status-<id>`, and `admin-user-save-<id>`;
- dynamic ID parts are normalized to safe lowercase HTML identifier fragments;
- internal elements generated by third-party libraries are not part of the application contract, but the frontend assigns IDs to the relevant application-owned containers and map markers.
